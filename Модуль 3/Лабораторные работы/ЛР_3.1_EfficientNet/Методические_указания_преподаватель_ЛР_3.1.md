# Методические указания для преподавателей по проведению ЛР № 3.1

## Тема и цель работы

**Тема:** "Реализация CNN для классификации изображений на TensorFlow/Keras. Сравнение различных API. Transfer Learning".

**Цель:** Знакомство студентов с методами переноса обучения (Transfer Learning) и дообучения (Fine-tuning) на примере классификации изображений цветов. Студенты изучают три способа построения моделей в Keras и сравнивают их по удобству реализации и качеству обучения.

## Знания, умения и навыки студентов

В ходе выполнения лабораторной работы студенты закрепляют и развивают следующие компетенции:

*   **DL-1.4 (Средний):** Разрабатывают и оптимизируют специализированные архитектуры для работы с изображениями (CNN) с использованием предобученных моделей. Студенты применяют Transfer Learning, заморозку и разморозку слоев, что позволяет анализировать влияние этих техник на качество классификации.
*   **MF-3.1 (Средний):** Применяют методы оптимизации для разработки и исследования обучающих алгоритмов. Сравнивая сходимость моделей и удобство разных API построения сетей, студенты развивают навыки выбора подходящего подхода в зависимости от задачи.
*   **PL-1.2 (Средний):** Осуществляют выбор и применяют инструменты разработки на Python (TensorFlow, Keras, NumPy, Matplotlib). Они оптимизируют код для обучения нейронных сетей с использованием библиотек для научных вычислений и работают с различными API фреймворка.

## Содержание работы

Лабораторная работа состоит из следующих этапов, позволяющих студентам постепенно перейти от теории к практике:

1.  **Подготовка данных:** Студенты загружают датасет FLOWERS, создают DataFrame, разделяют данные на обучающую и тестовую выборки, настраивают аугментацию данных и создают генераторы.
2.  **Часть 1: Sequential API:** Студенты реализуют модель с использованием Sequential API на базе EfficientNetB0, применяют Transfer Learning и обучают модель.
3.  **Часть 2: Functional API:** Студенты реализуют аналогичную модель, используя Functional API, и сравнивают результаты.
4.  **Часть 3: Subclassing API:** Студенты реализуют модель, используя Subclassing API, и сравнивают результаты.
5.  **Сравнительный анализ:** Студенты анализируют результаты обучения трех моделей, изучают влияние аугментации данных и делают выводы.

## Рекомендации по проведению

1.  **Демонстрация и повторение:**
    *   Проведите краткий обзор теоретической части Transfer Learning, объясните концепции заморозки слоев и дообучения.
    *   Разберите с аудиторией код для подготовки данных и создания генераторов. Убедитесь, что студенты понимают назначение аугментации данных.
    *   На примере Sequential API продемонстрируйте, как подходить к выполнению заданий.

2.  **Самостоятельная работа:**
    *   Предоставьте студентам достаточное количество времени (2–4 академических часа) на самостоятельное выполнение заданий по написанию кода, проведению экспериментов и анализу результатов.

3.  **Консультации и помощь:**
    *   Будьте готовы помогать студентам с отладкой кода, особенно при работе с памятью (batch size, размер изображений) и при реализации трех различных API. Обратите внимание на необходимость использования `GlobalAveragePooling2D` вместо `Flatten` для уменьшения количества параметров.

## Оценка выполнения работы

Оценка проводится на основе готового Jupyter Notebook, который студент сдает в конце занятия или в оговоренный срок. Критерии оценки детализированы в документе **«Критерии_оценки_ЛР_3.1.md»** (5-балльная шкала).

### Особые моменты оценки:

*   **Качество кода:** Обратите внимание на читаемость кода, наличие комментариев и структурированность ячеек.
*   **Глубина анализа:** Студент должен не просто запустить код, но и проанализировать полученные графики и дать развернутые ответы на контрольные вопросы. Важно, чтобы он сравнил три API по удобству/гибкости (а не ожидал большой разницы accuracy при одинаковой архитектуре) и обосновал выбор подхода под задачу.
*   **Полнота выполнения:** Задание должно быть выполнено полностью. Отсутствие ответов на вопросы или экспериментов с API значительно снизит итоговую оценку.
*   **Производительность:** Студенты должны продемонстрировать понимание ограничений по памяти и умение оптимизировать процесс обучения (выбор batch size, использование mixed precision).
