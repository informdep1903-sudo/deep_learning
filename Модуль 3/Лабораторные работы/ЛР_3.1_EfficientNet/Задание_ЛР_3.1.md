# Лабораторная работа № 3.1: Реализация CNN для классификации изображений на TensorFlow/Keras. Сравнение различных API. Transfer Learning

## Цель работы

*   Изучить методы переноса обучения (Transfer Learning) и дообучения (Fine-tuning) сверточных нейронных сетей для задачи классификации изображений на примере датасета FLOWERS (102 вида цветов).
*   Научиться реализовывать модели с использованием трех основных подходов в Keras: Sequential API, Functional API и Subclassing API.
*   Сравнить три способа построения моделей по удобству реализации и качеству обучения (при одинаковой архитектуре).
*   Изучить и применить на практике методы аугментации данных для улучшения обобщающей способности модели.
*   Закрепить навыки работы с библиотеками TensorFlow/Keras, Pandas, Matplotlib и другими инструментами для обработки данных и визуализации результатов.

## Задачи работы

В ходе выполнения лабораторной работы необходимо решить следующие задачи:

1.  **Подготовка данных:**
    1.  Загрузить и распаковать датасет FLOWERS (102 вида цветов).
    2.  Создать DataFrame с соответствием имен файлов и меток классов.
    3.  Выполнить разделение данных на обучающую и тестовую выборки (80/20) с сохранением баланса классов.
    4.  Настроить аугментацию данных для обучающей выборки (поворот, масштабирование, яркость, сдвиг, отражение).
    5.  Создать генераторы данных (`ImageDataGenerator`) для обучения, валидации и тестирования.
        *Важно:* для `tf.keras.applications.EfficientNetB0` не использовать `rescale=1./255` — модель ожидает пиксели в диапазоне `[0, 255]`.

2.  **Часть 1: Sequential API:**
    1.  Реализовать модель классификации с использованием Sequential API на базе предобученной EfficientNetB0.
    2.  Заморозить веса базовой модели для реализации Transfer Learning.
    3.  Обучить модель и сохранить историю обучения.
    4.  Визуализировать графики точности и функции потерь.

3.  **Часть 2: Functional API:**
    1.  Реализовать аналогичную модель, используя Functional API.
    2.  Сравнить процесс обучения и итоговую точность с Sequential моделью.

4.  **Часть 3: Subclassing API:**
    1.  Реализовать модель, используя Subclassing API (наследование от `tf.keras.Model`).
    2.  Сравнить процесс обучения и итоговую точность с предыдущими моделями.

5.  **Сравнительный анализ:**
    1.  Сравнить результаты обучения трех моделей.
    2.  Проанализировать влияние аугментации данных на качество классификации.
    3.  Сделать выводы о преимуществах и недостатках каждого из подходов (в первую очередь — удобство и гибкость API).

## Требования

*   **Инструменты**: Для выполнения работы необходимо использовать среду разработки с установленными библиотеками: Python 3.7+, TensorFlow 2.x, Keras, NumPy, Pandas, Matplotlib, Scikit-learn, Pillow.
*   **Код**: Код должен быть написан в формате Jupyter Notebook и содержать пояснительные Markdown-ячейки.
*   **Выполнение**: Все части работы (создание моделей, обучение, анализ) должны быть реализованы и запущены.
*   **Аппаратные требования**: Для обучения EfficientNetB0 рекомендуется использовать GPU (например, Google Colab с GPU T4).
